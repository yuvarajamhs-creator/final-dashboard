Ads Analytics Dashboard — Hook Rate & Hold Rate Methodology

1. Data source and scope

AspectDetail

2. Hook Rate

2.1 UI description

The card subtitle is: 3s views ÷ impressions · else plays ÷ impressions — three-second views are preferred; if Meta returns no 3-second volume for the aggregate, the app uses video plays ÷ impressions.

2.2 Formula (KPI totals)

Let I = sum of impressions, V3s = sum of 3-second video views, P = sum of video plays.

Hook Rate (%) =
• If V3s > 0: (V3s / I) × 100
• Else if P > 0: (P / I) × 100
• Else: 0  (also if I = 0)

Meta fields (typical): video_3_sec_watched_actions (often top-level on the insight row), plus fallbacks in actions; plays from video_play_actions / video_play / video_view.

2.3 Example — Hook Rate 94.83% (e.g. ad “Ladu”, ~215,096 impressions)

If the 3-second path is active: 0.9483 ≈ V3s / 215,096 → V3s ≈ 203,900 (order-of-magnitude check).

If only the plays fallback were used (V3s = 0), the same percentage would mean plays are a similar fraction of impressions — verify in Meta which metrics are non-zero for that ad and date range.

3. Hold Rate

The UI may say “100% Watched / Video Play”; the implemented KPI uses a priority cascade (see code). Summed inputs include 3-second views, p50/p75 watched, ThruPlay, 100% watched, plays, and video views.

3.1 Hold numerator H (first positive wins)

  1. video_p50_watched_actions if > 0
  2. Else video_p75_watched_actions if > 0
  3. Else video_thruplay_watched_actions if > 0
  4. Else 100% / completion count

3.2 KPI cascade

1) If V3s > 0 and H > 0:   Hold Rate = (H / V3s) × 100

2) Else if plays > 0 and P100 > 0:   Hold Rate = (P100 / plays) × 100

3) Further fallbacks (ThruPlay vs plays, etc.) — see aggregateHoldRatePct in the codebase.

3.3 Example — Hold Rate 1.08%

If branch (1) applies: 0.0108 ≈ H / V3s, so H is about 1.08% of V3s — a small share of 3-second viewers reaching the milestone used as H. If branch (2) applies, interpret as ~1.08% of plays with 100% watched.

4. Filters

Selections for ad account, campaign, and ad name restrict which insight rows are summed. Hook and Hold always match the current filter scope and date range.
